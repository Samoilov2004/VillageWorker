ML-микросервис для платформы цифрового хаба сельских территорий.

## Назначение

Сервис реализует интеллектуальную обработку данных и предоставляет API для следующих задач:

- рекомендации похожих вакансий, резюме и других сущностей;
- пре-модерация пользовательского контента;
- поисковая выдача и ранжирование объявлений;
- хранение и обновление внутреннего индекса сущностей, используемого для поиска и рекомендаций.

Сервис является отдельным компонентом архитектуры и вызывается backend'ом по HTTP API.

Все ML-методы публикуются под префиксом `/api/ml`. Примеры:
- `/api/ml/index/upsert`
- `/api/ml/search`
- `/api/ml/recommend/similar`
- `/api/ml/moderation/check`

Служебные методы доступны отдельно:
- `/health`
- `/ready`

## Запуск микросервиса
```bash
python3 -m venv .mlvenv
source .mlvenv/bin/activate
pip install -r ml_service/requirements.txt

uvicorn ml_service.app.main:app --reload --host 127.0.0.1 --port 8001
```

## Запуск тестов
```bash
pytest ml_service/tests -v
```

## Документация API
После запуска микросервиса будет доступна встроенная техническая документация API и служебные точки проверки состояния сервиса.

### Swagger UI
`http://127.0.0.1:8001/docs`

Swagger UI — это интерактивная веб-страница с документацией по всем доступным endpoint'ам микросервиса.  
Она автоматически формируется FastAPI на основе кода приложения и описанных схем запросов/ответов. Это основной способ быстро проверить, как работает API, и понять, как backend должен к нему обращаться.

### ReDoc
`http://127.0.0.1:8001/redoc`

ReDoc — это альтернативное представление документации API.  
В отличие от Swagger UI, он меньше ориентирован на интерактивное тестирование и больше подходит для чтения структуры API в удобном и более “справочном” виде.

### Health-check
`http://127.0.0.1:8001/health`

Health-check — это служебный endpoint для быстрой проверки, что микросервис запущен и отвечает на запросы.
Если сервис работает корректно, запрос к `/health` возвращает простой ответ вида:

```json
{
  "status": "ok"
}
```
